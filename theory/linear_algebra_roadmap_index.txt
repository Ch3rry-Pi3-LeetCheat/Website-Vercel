Linear algebra roadmap index

Purpose of this document

This document defines the learning roadmap for a linear algebra theory library.
It is intended to act as an index for a structured set of lesson notes, moving
from complete beginner ideas to advanced topics.

The goal is not to write all of linear algebra in one file.
The goal is to define the order in which the theory should be learned and documented,
so that each lesson builds naturally on the previous one.

This document should therefore be treated as a roadmap and index, not as the final notes.

How this will be used

The plan is to work through linear algebra topic by topic.
For each topic listed below, a separate beginner-friendly lesson note can later be created.

Each future lesson note should aim to include:

- intuitive explanation
- geometric intuition where useful
- formal definitions
- worked examples
- common mistakes and misconceptions
- computational examples
- why the topic matters
- links to the previous and next lessons

This means the linear algebra folder can gradually become a structured library
of learning notes rather than a random collection of disconnected topics.

Why this roadmap exists

Linear algebra becomes difficult when topics are learned in the wrong order.
For example, eigenvalues make little sense without matrices and linear transformations.
Least squares is much easier once projections and orthogonality are clear.
SVD is far easier once basis, rank, diagonalization, and orthonormality are in place.

This roadmap is therefore designed as a dependency-aware study sequence.

Recommended first block

The best first block is:

- scalars, vectors, and vector notation
- vector addition and scalar multiplication

- linear combinations and span
- linear independence, basis, and dimension

This block creates the conceptual foundation needed before matrices are introduced.

Working conclusion

This roadmap should be treated as the master index for a structured linear algebra library.
The key idea is simple:

learn the ideas in the right order, document them clearly, and make each later topic
easier by building on what has already been understood.
